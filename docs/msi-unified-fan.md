# MSI 笔记本风扇统一联动（msi-unified 分支）

本分支为 THRM 新增 **MSI 笔记本 EC 风扇直控**与**统一热管理联动**：把飞智散热器
和 MSI 笔记本的 CPU/GPU 双风扇当作一个整体协调控制。

> 目前适配机型：**MSI Vector 16 HX A14VHG**（gen-2 "IsNewEC" EC，固件 15M1IMS2.111）。
> 同代 MSI 机型（Raider/Vector/Katana 2023+ 等）寄存器布局相同，理论兼容，
> 但未经验证——启用前请自行确认（后端会校验 EC 固件版本特征，不识别则拒绝启用）。

## 工作原理

### 笔记本侧：EC 曲线直写（不是软件实时调速）

MSI 的 EC 固件本身就按"温度→转速曲线"自动控制风扇。本功能通过 WinRing0
驱动把**整条曲线**写入 EC（7 个速度点 + 6 组升/降温阈值），之后由 EC 硬件自主
执行。这带来一个关键安全属性：**即使 THRM 崩溃或被杀掉，风扇依然按最后写入
的曲线正常控温**，不存在"软件死了风扇停转"的风险。

内置三条预设曲线（速度值为 EC 原生单位，0–150）：

| 预设 | 行为 |
|---|---|
| Silent | 低温区风扇完全停转，中温区维持低转速 |
| Default | 机型出厂默认曲线（社区验证值） |
| Aggressive | 各温度点提前、加强，上限 120（为 Cooler Boost 留余量） |

### 统一联动算法（unifiedfan）

每个温控 tick（默认 2 秒）：

1. **需求计算**：CPU/GPU 各自把温度映射为 0–1 的散热需求（CPU 45→92°C，
   GPU 45→85°C），并叠加升温速率前馈（快速升温时提前加大需求）。
2. **散热器减负积分**：散热器出力越大（当前转速/满速比例越高），笔记本风扇
   需求越低——压风散热器安静、风量大，优先让它出力，笔记本风扇（噪音源）
   滞后介入。温度达到守卫值（CPU 85°C / GPU 80°C）后积分作废，不拿散热
   换安静。
3. **曲线混合**：按最终需求在 Silent→Default→Aggressive 之间线性混合出一条
   曲线写入 EC（带量化去抖，只有曲线真正变化才写寄存器）。
4. **panic 直通**：CPU ≥95°C 或 GPU ≥87°C **连续两个 tick**（过滤瞬时尖峰）
   → 开启 Cooler Boost（EC 满速）+ 散热器满速直通；降回 88/81°C 以下解除。
5. **失援补偿**：散热器断连/未接时，笔记本曲线至少回到 Default。

散热器自身转速仍由 THRM 原有管线（用户曲线 + 自适应学习 + 功耗前馈 +
转速避让）决定，联动不改变原有行为。

### 安全设计

- **高温尾部保护**：无论算法输出什么，曲线最高温的 2 个速度点永远不低于
  出厂默认值（`msifan.ClampToSafe`）。
- **EC 校验**：初始化时读取 EC 固件版本（0xA0 处 12 字节 ASCII，须含 "IMS"
  特征）+ 风扇模式寄存器合法性校验，不认识的机器拒绝写入。
- **退出还原**：THRM 正常退出/关闭联动时，自动恢复默认曲线、关闭 Cooler
  Boost 并卸载驱动。
- **panic 需连续确认 + 迟滞恢复**，避免误触发/振荡。

## 使用步骤

1. 确保 `WinRing0x64.sys` 与 `THRM Core.exe` 同目录（`build.bat` 已自动获取；
   手动构建可运行 `python3 scripts/fetch_winring0.py <输出目录>`）。
2. 以管理员运行 THRM（core 的 manifest 本身要求管理员）。
3. 设置面板 →"MSI 笔记本风扇"→ 打开"启用 MSI EC 控制"。
4. 打开"统一联动"即为全自动；关闭联动则仅监控转速/温度，并可手动开关
   Cooler Boost。

## Windows Defender / 内存完整性注意事项

WinRing0 是 2007 年的老牌签名驱动，因历史漏洞（任意端口 IO）被微软列入
易受攻击驱动名单：

- **内存完整性（HVCI）开启时驱动会被拦截**：设置 → Windows 安全中心 →
  设备安全性 → 内核隔离 → 关闭"内存完整性"（YAMDCC/OpenRGB 等所有
  WinRing0 系工具同此要求）。
- Defender 可能报告 `WinRing0x64.sys` 为 HackTool/风险项，需添加排除或
  允许。它的"风险"即"允许管理员程序直接访问硬件端口"——这正是 EC
  控制所需的能力，风险自担。

## 配置项（config.json）

```json
"msiEcFan": {
  "enabled": false,      // 启用 MSI EC 后端（默认关）
  "linked": true,        // 统一联动；false = 仅监控
  "driverPath": ""       // WinRing0x64.sys 路径；空 = core 程序目录
}
```
